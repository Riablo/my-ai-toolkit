# Image Prompt Tester

用于快速验证图片 prompt 的实际效果，或从参考图抽出可复用的风格模板。

## 两种模式

- 提示词测试：用户给 prompt 或场景描述，想实际出几张图对比
- 参考图风格测试：用户给图片，想提取风格并得到带占位符的模板

## 核心原则

- 默认测试 3 张或 3 个版本，除非用户指定数量
- 不要过度改写原 prompt；只围绕测试方向做必要变化
- 如果用户没说清测试方向，先问一个简短问题；用户说“你定”时再自行决定
- 多图必须按“一张图一个子任务”并行生成，不要串行出图
- 每组测试都保存到 `~/Downloads` 下的新目录，并把图片、最终 prompt、模板和 manifest 一起落盘

## 提示词测试

1. 提取原 prompt、测试方向和数量
2. 若方向不明，先追问或给 2-4 个可选方向
3. 每个候选 prompt 只改一个主要维度，避免把原画面改写成另一套东西
4. 用户要原版对照时，再把原 prompt 作为一个候选

## 参考图风格测试

1. 提取媒介、构图、光线、色彩、材质、镜头、氛围和明显限制
2. 默认生成 3 个带占位符的模板，如 `{主体}`、`{场景}`、`{动作}`
3. 每个模板用一组具体填充值出图验证，但最终交付应保留占位符版本
4. 生成后做一次简短视觉复核，指出最明显的优点和偏差

## 输出目录

目录建议：

```text
~/Downloads/image-prompt-test-YYYYMMDD-HHMMSS-<slug>/
```

至少保存：

- `original-input.txt`
- `manifest.md`
- `prompts.json`
- 每张图对应的 `XX-prompt.txt`
- 每张生成图

参考图测试还应保存模板文件；做了复核时再补 `evaluation.md`。

## 高价值 gotchas

- 如果参考图里有书法、印章、招牌等文字元素，不要强求模型生成可读文字；更适合写成装饰性、模糊、低存在感的文字感元素
- 复刻传统绘画时，如果结果总跑偏到现代插画或高清写实，需要显式压低对比、体积光和数码质感
- 图片必须真正保存成功；如果只能保留 URL，要在 manifest 里说明

## 最终回复

- 输出目录
- 每个版本的图片路径或展示图
- 每张图的最终 prompt
- 若是参考图测试，再补模板和本次填充值
- 失败项和失败原因
